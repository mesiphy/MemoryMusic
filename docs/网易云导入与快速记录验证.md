# 网易云导入与快速记录验证

> 验证日期：2026-07-15—16<br>
> 对应阶段：M4（平台无关导入与快速记录已完成；真实账号导入待授权）<br>
> 成品环境：Windows 11 Pro x64 `10.0.26200`、Electron `39.8.10`、网易云音乐 `3.1.36.205322`

## 1. 官方能力核验边界

本阶段只评估网易云官方开放平台与官方工具，不引入私有 API、抓包接口、Cookie、桌面客户端凭据或播放破解。

截至核验日，[网易云官方技能仓库](https://github.com/NetEase/skills)、[官方 `@music163/ncm-cli` 包](https://www.npmjs.com/package/@music163/ncm-cli)、[个人开发者 FAQ](https://developer.music.163.com/st/developer/document?docId=3b75ab8e475d41ca93d91ebd4dfd383f)及开放平台公开文档表明：

- 当前核验版本为 `@music163/ncm-cli@0.1.6`，要求 Node.js 18 或更高版本；
- 首次使用需要开放平台 App ID 与私钥，并通过 `ncm-cli login` 扫码授权；
- 个人开发者目前只允许使用官方 `ncm-cli`，不能直接调用开放平台 HTTP API；公开 HTTP 文档只能用于核验能力和字段，不能作为本项目个人场景的生产接入方式；
- 官方 CLI Skills 要求先运行 `ncm-cli commands` 获取当前动态命令树，再通过命令级 `--help` 核对参数，并明确要求不得猜测参数；
- 官方音乐助手 Skill 明确使用 `playlist collected` 获取红心数据，最多分析 200 首、`index=0` 为最新，条目包含 `extMap.addTime`（毫秒）、`songTag[]`、`artists[]`、`album`、`name` 与 `duration`；
- 动态命令的完整参数、输出包装和账号实际权限依赖有效凭据与登录态，不能从未授权帮助信息中可靠推断。

### 1.1 公开文档交叉核验（2026-07-16）

开放平台公开文档进一步确认了官方能力，但不会被应用直接调用：

- [获取用户红心歌单](https://developer.music.163.com/st/developer/document?docId=f0b639bf1494424188a8360d4a22fdd4)返回“我喜欢的音乐”歌单 ID，并明确要求 `accessToken`；
- [获取歌单里的歌曲列表 v3](https://developer.music.163.com/st/developer/document?docId=89ad91a1040b477e9f4d7bea2ea08163)使用 `limit`/`offset`，单页最多 500 首、建议 30 首，返回歌曲 ID、名称、时长、艺人、专辑、标签、可见性等字段；
- [获取歌单里的歌曲列表 v4](https://developer.music.163.com/st/developer/document?docId=781b8d60178f428695e8ff3b464a3736)最多返回 11000 首歌曲的 ID 与 `addTime`，与官方 Skill 中的收藏时间能力相互印证；
- [个人入驻与使用引导](https://developer.music.163.com/st/developer/document?docId=9504d35aa41a47c6ac9830b2dbf48f94)说明每日调用次数有上限，具体剩余次数只在登录后的应用详情中展示，公开资料没有给出可据以验收的固定个人额度。

因此生产适配器继续保持关闭。直接照搬上述 HTTP URL 会违反官方个人接入边界；仅凭 `playlist collected` 和字段说明实现输出解析，又会猜测登录后动态下发的包装结构。正确的下一步是在用户完成外部授权后，依次记录 `ncm-cli commands`、`ncm-cli playlist collected --help`（若授权命令树确认该路径）和一份脱敏真实输出，再固定进程参数与解码器。

本地执行 `pnpm dlx @music163/ncm-cli@0.1.6 --help` 时，官方 CLI 明确要求先配置 API key。随后从 npm 官方包重新解包并在独立 HOME 中安装其声明依赖；`--help`、`commands`、`playlist --help`、`playlist collected --help` 和 `login --check` 在无凭据状态下均以同一 API key 前置错误退出，证明登录前无法读取动态契约。隔离审计目录中只写入过虚假 App ID/私钥；CLI 生成了加密的 `credentials.enc.json`，文件中未出现虚假私钥明文。虚假凭据同样无法加载登录后的动态用户命令。所有审计目录均在核验后删除。

尚未验证、也不在本文中宣称已验证的内容：

- 真实扫码登录、歌曲搜索和歌曲 ID 响应；
- “我喜欢的音乐”权限、分页方式、配额与收藏时间字段；
- 真实账号至少 100 首收藏的首次导入和重复同步。

## 2. 平台无关导入闭环

`NeteaseDataAdapter` 只向领域层暴露已归一化的歌曲、不可见歌曲 ID、下一页游标和增量检查点。领域层不依赖 CLI/HTTP 原始响应结构，也不会保存原始响应。

SQLite v4 为 `sync_state` 增加以下持久状态：

- `status`：`idle`、`running`、`succeeded` 或 `failed`；
- `last_attempt_at` 与 `last_success_at`；
- `failure_reason` 与 `retry_count`；
- 可恢复的 opaque `cursor`。

每页歌曲与游标在同一事务中提交。失败后保留最后成功页的游标，重试从该游标继续；未知异常只保存固定错误文案，不把可能含凭据的异常正文写入数据库或日志。

导入写入规则：

- 新平台 ID 优先复用同名同歌手、且尚无网易云映射的手工歌曲，否则创建新歌曲；
- 已存在映射只更新 `provider_tracks` 的 URL、可用状态、最后发现时间和白名单元数据；
- 不更新既有歌曲标题、歌手、专辑或时长，更不覆盖标签、感悟、事件、别名和其他召回线索；
- 不同平台 ID 即使歌名和歌手相同也保持为不同歌曲，避免错误合并版本；
- 平台删除或不可见只把映射标记为不可用，不删除歌曲或个人资料。

自动化证据：

- 125 首 Mock 收藏分 3 页完成导入，其中 124 首新建、1 首复用手工歌曲；
- 重复同步 125 个映射时歌曲总数仍为 125，个人标签、感悟、事件和别名逐项保持；
- 第二页失败后保存 `retry-page` 与失败原因，下一次从该页继续并清零重试状态；
- 不可见歌曲只更新映射状态；两个同名同歌手但平台 ID 不同的条目不会被错误合并；
- 官方适配器不可用时，既有资料仍可离线编辑与搜索。

生产环境当前使用显式 unavailable 适配器；这是有意的安全降级，不会在缺少真实授权证据时猜测动态命令或改用非公开接口。

## 3. Windows Credential Manager

主进程实现了 `NeteaseCredentialStore`：

- 使用 Win32 `CredWriteW`、`CredReadW`、`CredDeleteW` 访问 Generic Credential；
- PowerShell 子进程命令行只包含固定脚本，秘密 JSON 只通过标准输入传递；
- stdout 仅在读取时回传给主进程，错误结果不包含秘密或原始异常；
- SQLite、渲染层持久状态和日志均不保存 App ID/私钥；
- 非 Windows 平台返回明确的不支持错误。

真机集成测试为每次运行创建唯一的 `MemoryMusic/Test/<UUID>` 目标，写入虚假 App ID/私钥，读取比对后在 `finally` 中删除。验证后 `cmdkey /list` 中 `MemoryMusic/Test/` 残留目标数为 0。没有访问、迁移或修改用户的真实网易云凭据。

运行真实安全存储测试：

```powershell
$env:MEMORY_MUSIC_WINDOWS_CREDENTIAL_TEST='1'
pnpm test -- src/main/__tests__/windows-credential-manager.integration.test.ts
```

## 4. 快速记录与待整理箱

`Ctrl+Shift+M` 注册为全局快捷键，并打开使用同一 preload 安全边界的紧凑窗口。保存时主进程重新读取 SMTC 当前歌曲，按歌名和歌手复用或创建本地歌曲，然后执行三种最短路径之一：

- 标签：输入一个标签并保存；
- 一句话：追加一条带时间感悟；
- 稍后整理：允许零文本保存到 `quick_capture_inbox`。

待整理箱在主窗口中列出未处理记录并支持标记已整理。主窗口重新获得焦点时自动刷新资料库与待整理箱。保存失败时小窗保持用户输入并显示明确错误；只有主进程确认成功后才清空并关闭。

### 4.1 隔离成品真机结果

成品以独立 `--user-data-dir` 启动，网易云原媒体状态为“沦陷 / 徐秉龙 / 已暂停”。验证结果：

- 从网易云窗口按下全局快捷键能打开快速记录小窗；
- 小窗正确显示“沦陷 / 徐秉龙”，状态为媒体会话已连接；
- 一句话、标签 `M4真机标签`、零文本待整理三条路径均成功；
- 感悟和标签均可通过统一搜索命中，待整理记录在主窗口刷新后可见；
- 5 次完整“快捷键 → 输入 → 保存 → 小窗关闭”耗时分别为 `1733 / 1744 / 1693 / 1901 / 1699 ms`，中位数 `1733 ms`，低于 10 秒标准；
- 小窗保存并关闭后，主窗口获得焦点即自动刷新并显示新歌曲详情，无需重启或手动刷新；
- 重启成品后歌曲和个人记录仍可搜索；数据库为 schema v4，`integrity_check=ok`，包含 1 首歌曲、1 个标签、6 条感悟和 1 条待整理记录；
- 验证后关闭成品、删除隔离用户目录，网易云仍为原歌曲且保持暂停。

## 6. 当前外部阻塞与继续条件

真实收藏导入必须由用户完成以下外部步骤，不能由自动化代替：

1. 由成年实名用户完成网易云开放平台个人入驻和应用初始化，在管控台取得有效 App ID 与私钥；不要把私钥粘贴到聊天、Git、日志或数据库。
2. 在本机隔离终端中按官方 README 配置 `@music163/ncm-cli@0.1.6`，再执行 `ncm-cli login`，使用目标网易云账号扫码授权；这些身份确认和扫码动作必须由用户本人完成。
3. 授权后先运行 `ncm-cli login --check` 与 `ncm-cli commands`。不要在聊天中发送凭据；只需确认登录检查通过，并允许继续在本机读取已授权的命令级帮助和脱敏输出。
4. 该账号应具有至少 100 首“我喜欢的音乐”，以完成真实分页/数量边界、实际配额、收藏时间、幂等和不可见退化验收。

授权完成后，继续工作将先读取官方 CLI 的已授权动态帮助与 JSON 样例，固定生产适配器和字段映射，再在隔离数据库中执行真实 100 首导入。未完成这些步骤前，M4 不标记整体完成，也不进入依赖 M4 的 M5。
