# M5 自用验证记录指南

本文定义 MemoryMusic 四周真实自用阶段的任务口径、记录字段和复盘方法。真实查询、目标歌曲和个人记忆只写入 `.local/m5-self-use-log.md`，该目录已被 Git 忽略；仓库只保留方法，不提交个人内容。

## 1. 验证周期与目标

- 周期：2026-07-20 至 2026-08-16，共四周；
- 最终复盘：2026-08-17；
- 样本：至少 20 个真实忘歌任务；
- 核心目标：至少 80% 的任务在 10 秒内找回目标；
- 辅助指标：找回率、无结果率、记录耗时中位数、播放失败次数和同步失败次数。

四周内不因为个别失败立即加入查询语法、时间线或语义搜索。先记录失败原因；只有数据丢失、安全问题或阻断日常使用的缺陷可以中断观察期。

## 2. 什么算真实忘歌任务

满足以下条件才计入样本：

1. 当时确实想找一首特定歌曲；
2. 开始时不知道或不能确定歌名和歌手；
3. 使用当时自然记得的线索搜索，不先查看网易云收藏或测试夹具；
4. 从开始输入第一个查询计时，到确认目标歌曲或决定放弃为止；
5. 无论成功、失败或超过 10 秒都记录，不能只保留成功样本。

为了凑数而预先挑歌、照着歌曲资料编线索，或重复同一个任务，不计入 20 个真实样本。

## 3. 每次需要记录什么

忘歌任务至少记录：

- 日期时间；
- 当时自然记得的线索和第一次查询；
- 找回后确认的目标歌曲；
- 是否找回、耗时、结果排名、尝试次数和无结果次数；
- 帮助找回的命中字段，或缺失但可能有用的个人字段；
- 从搜索结果发起播放是否成功，是否使用网页兜底。

主动标注歌曲时另记入口、内容类型、耗时和是否成功。同步失败、播放失败、保存失败和应用异常单独记入故障表。

## 4. 每周复盘

| 周次 | 日期 | 最低动作 |
| --- | --- | --- |
| 第 1 周 | 07-20 至 07-26 | 熟悉记录方式，不改搜索算法 |
| 第 2 周 | 07-27 至 08-02 | 备份后清理重复标签、无效事件和错误关联 |
| 第 3 周 | 08-03 至 08-09 | 检查失败是否集中在同一种缺失线索 |
| 第 4 周 | 08-10 至 08-16 | 补足真实样本，不制造测试任务 |
| 最终复盘 | 08-17 | 计算指标并决定继续、缩减或停止 |

周度复盘计算：

- 10 秒成功率 = 10 秒内找回的真实任务数 ÷ 真实任务总数；
- 总找回率 = 最终找回的任务数 ÷ 真实任务总数；
- 无结果率 = 至少出现一次无结果的任务数 ÷ 真实任务总数；
- 记录耗时取中位数，不用平均值掩盖极慢操作；
- 同类失败达到可观察频率后，才提出字段或功能改动。

## 5. 数据安全

- 每周清理个人资料前先执行一次手动备份；
- 不编辑或覆盖 `memory-music-initial-791-verified.sqlite3` 基线快照；
- 不把 `.local/`、真实查询、账号信息、CLI 输出或凭据加入 Git；
- 自动备份不替代周期开始时的独立快照；
- 发现数据损坏时停止写入，保留主库、WAL/SHM 和最近备份后再诊断。
